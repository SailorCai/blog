# docker-compose

一个常规的前后端分离项目，至少是包含服务端和前端两个项目的，甚至像nodejs全栈项目可能还包含数据服务，redis 存储服务等，这么多服务总不能自己一个个手动跑，docker-compose就是来解放你的双手的，他让你可以一次编写，持续集成，想在哪部署只需执行一下docker-compose up即可一键启动所有服务，是不是很方便？
