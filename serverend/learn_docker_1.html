# docker 入门

这里目前是一些自己学习的笔记

# 一些常用常用命令

  • 定制一个镜像
docker build -t nginx:latest .

命令解释：

-t nginx:latest: 表示指定镜像的名称和 tag，这里定制的镜像名称叫 nginx，tag 是 latest

.: 代表依据当前目录的 Dockerfile 定制镜像

  • 启动一个镜像并指定容器的 name
docker run -p 3000:7777 --name blog_hook -d nginx:latest

命令解释：

-p 3000:7777: 指定宿主机和容器的端口映射，3000 代表宿主及端口，7777 代表容器暴露的端口

--name: 为运行镜像的容器指定名称，这样后面就能通过名称操作容器，比如docker stop blog_hook，而不需要使用<containerId>

-d: 表示启动一个 daemon 实例，em...我也不知道 daemon 实例是啥，简单说就是加了-d后镜像会在后台启动

nginx:latest：这个就是你上面定制镜像是指定的镜像名称和 tag（或者版本号）

  • 启动已有容器
docker run <containerId> | <containerName>
  • 查看运行中的容器
docker ps
  • 查看所有容器
docker ps -a
  • 删除指定容器
docker rm <containerId>
  • 同时删除多个状态为“exited”的容器
docker rm $(docker container ls -f "status=exited" -q)
  • 删除所有容器
docker rm $(docker container ls -aq)
  • 进入某个容器的命令行
docker exec -it <containerId> sh
  • 查看某个容器的日志（实时跟踪）
docker logs -f <containerId>

# 定制一个 nginx 镜像

Dockerfile 内容如下：

FROM nginx:latest

RUN echo '<h1>hello docker !!!</h1>' > /usr/share/nginx/html/index.html

命令解释：

FROM nginx:latest: 以镜像库 nginx 最新版本来构建当前镜像

RUN: 执行命令，其实就是输出了一段 html 内容到容器的 /usr/share/nginx/html/index.html 中
